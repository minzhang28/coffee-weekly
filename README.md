# Coffee Weekly Reports

Automated weekly coffee bean recommendations for Vancouver's Chinese coffee community.

## How It Works

1. **Apps Script** generates bilingual markdown reports (Chinese + English)
2. **Apps Script** pushes markdown files to this repository
3. **GitHub Actions** automatically converts markdown to HTML
4. **HTML files** are committed back to the repository

## Repository Structure

```
posts/
└── YYYY-MM-DD/
    ├── chinese.md       (Generated by Apps Script)
    ├── chinese.html     (Generated by GitHub Actions)
    ├── english.md       (Generated by Apps Script)
    └── english.html     (Generated by GitHub Actions)
```

## Viewing Reports

- Browse to `posts/YYYY-MM-DD/chinese.html` or `posts/YYYY-MM-DD/english.html`
- Open in browser to see the styled weekly report

## Files in This Repo

- `.github/workflows/convert-markdown.yml` - GitHub Actions workflow
- `convert.js` - Markdown to HTML conversion script
- `template.html` - HTML design template (neumorphic style)
- `package.json` - Node.js dependencies (marked, cheerio)

## Automatic Conversion

When markdown files are pushed to `posts/**/*.md`:
1. GitHub Actions triggers automatically
2. Installs dependencies (`marked`, `cheerio`)
3. Runs `convert.js` to generate HTML
4. Commits HTML files back to the repository

## Manual Conversion (Local Testing)

```bash
npm install
node convert.js
```

This converts all markdown files in the `posts/` directory.
